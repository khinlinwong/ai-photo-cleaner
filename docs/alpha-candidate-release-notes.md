# AI Photo Cleaner 内部 Alpha Candidate 发布说明 (Release Notes)

本说明文档面向内部测试人员，详细说明 **AI Photo Cleaner** 桌面端首个内部 Alpha 候选包的基线、安装指引、测试重点及已知风险。

---

## 1. 软件基线与包体信息 (Baseline & Packages)

*   **当前代码基线**: `77c90b8` (含 A/B 多张相似照片对局败者淘汰路由修复，即 fix ab multi photo loser routing)
*   **主分发安装包 (主推)**: 
    *   **文件名**: `AI Photo Cleaner_0.1.0_x64-setup.exe` (NSIS 安装包)
    *   **文件大小**: 约 `2.59 MB` (精确大小：2,719,529 字节)
    *   **定位**: 内部 Alpha 测试的主力分发包，集成了已修复的覆盖安装、卸载控制流模板，A/B 对局防跳边体验优化，以及最新的多张组败者淘汰路由逻辑。
*   **辅助验证包 (不作分发)**:
    *   **文件名**: `AI Photo Cleaner_0.1.0_x64_en-US.msi` (MSI 安装包)
    *   **文件大小**: 约 `3.66 MB` (精确大小：3,837,952 字节)
    *   **定位**: 仅作为打包编译的完整性验证，**测试人员请勿安装或作为主测试包**。
*   ⚠️ **旧包作废声明**: 之前的代码基线 `1188a4c` 对应的旧安装包已正式作废。该旧包中存在“3张及以上相似组对局时未被选择保留的 loser 照片错误残留在 Keep 区”的逻辑 Bug。测试人员必须仅使用当前基于 `77c90b8` 重新生成的 NSIS 安装包进行测试。
*   ⚠️ **重要警告**: **禁止 NSIS 与 MSI 混用**！混用会导致安装目录文件覆盖及系统注册表写入冲突，造成软件状态异常。
    *   **不要运行 MSI 安装包**，请统一部署和分发 NSIS 格式的 `setup.exe`。

## 2. 安装说明 (Installation Guide)

1.  **启动安装**: 在 Windows 环境下双击运行 `AI Photo Cleaner_0.1.0_x64-setup.exe`。
2.  **安全提示处理**: 
    *   由于目前尚未导入正式的代码签名证书，Windows 系统会弹出 **SmartScreen 拦截提示** (未知发布者)。
    *   请在弹出框中点击 **“更多信息”**，然后选择 **“仍要运行”** 即可进入正常安装向导。
3.  **杀毒与安全**: 经 Windows Defender 及主流安全检测，此包安全无毒，无任何安全阻碍。
4.  **启动应用**: 安装向导执行完成后，可在最后一页勾选“运行 AI Photo Cleaner”，或从 Windows **“开始菜单”** 的快捷方式启动 App。

## 3. 测试者建议测试流程 (Recommended Test Flow)

我们建议您按照以下操作路径对产品进行基本功能的覆盖测试：

1.  **TC-01: 30 张选择图片模式**
    *   进入桌面首页，选择“选择图片”模式，导入 30 张左右本地照片（建议包含连拍或相似的图片）。
    *   点击“开始本地分析”，等待其进入整理结果页（Results）。
2.  **TC-02: 选择文件夹模式**
    *   返回首页，使用“选择文件夹”模式，选择包含多张照片的本地目录，开始分析。
3.  **TC-03: Results 整理结果页操作**
    *   检查 Keep (保留) 与 Cull (淘汰候选) 照片卡片的区分度（浅绿/浅红边框和底色）。
    *   鼠标悬浮于图片上时，观察是否浮现带小放大镜的“预览大图”提示。
    *   点击图片、勾选 Checkbox、使用底侧“标记为保留/淘汰候选”按钮，确认逻辑和响应是否正常。
4.  **TC-04: A/B 相似照片对比**
    *   若分析出了相似组，观察是否自动弹窗进入 A/B 对局页面。
    *   测试鼠标滚轮对左右图进行缩放、左键按住拖拽平移、双击重置缩放位移。
    *   使用键盘快捷键进行快速筛图决策：
        *   `←` / `→`: 选择保留左图/右图。
        *   `B`: 保留两张。
        *   `C`: 两张都标记为淘汰候选。
        *   `S`: 跳过当前组。
        *   `Esc`: 退出/关闭 A/B 对局弹窗。
5.  **TC-05: 卸载测试**
    *   测试通过 Windows 系统的“设置 - 已安装的应用”或控制面板卸载应用，确认快捷方式和程序文件干净清除，且不会发生异常自动重新安装。

## 4. 当前已验证能力 (Verified Features)

本安装包已通过最终的 Smoke QA 流程，具体验证通过项如下：

*   **安装器及维护模式**: 双击正常启动并完成安装；Windows Apps 卸载条目完美清除无残留，卸载后不发生自动重新安装；维护界面仅卸载不重装、Add/Reinstall 覆盖修复逻辑完全通过验证。
*   **A/B multi-photo loser routing 逻辑修复与验证**:
    *   修复了在 3 张及以上的多张照片相似组对局中，中途被淘汰的照片未进入“淘汰候选”的 Bug。
    *   现在每轮被淘汰的照片（loser）会立即移出 `newRecommended`（建议保留）和 `newBackup`（备选）列表并推入 `newCull`，避免同一照片 ID 同时处于 Keep 和 Cull 的冲突状态。
    *   完成了全流程逻辑模拟验证，通过了 2张组、3张连续同侧保留、3张中间变动胜者、3张 champion“先赢后输”、4张以上复杂对局、Keep Both、Cull Both、Skip 各大场景的验证，保证最终只建议保留唯一的 champion（除非选了“保留两张”）。
*   **A/B winnerSide 体验修复**: 
    *   在 A/B 挑选过程中选择右侧照片胜出，胜出照片在下一轮继续保留在右侧，新挑战照片在左侧展示，不产生跳边。
    *   选择左侧照片胜出，胜出照片在下一轮继续留在左侧，新挑战照片在右侧展示。
    *   支持连续多次同一侧胜出及左右交替胜出，逻辑和展现均保持连贯。
    *   移除全部胜负/挑战者切换动画（swapping / animate-battle-challenger-in），实现 0 延迟即时无闪烁切图。
    *   图片区域只支持平移、滚轮缩放、双击重置，单击图片本身不触发任何决策动作，完全避免了看图误触。
    *   快捷键 (← / → / B / C / S / Esc) 继续保持物理位置对应决策，即时响应无延迟感。
    *   切换至新相似组时，位置状态成功重置，上一组的 `winnerSide` 不污染新组。
*   **扫描与数据脱敏**: 30 张基础测试通过，扫描能正常走完并自动流转。文件名和物理路径保持脱敏，仅显示 Photo-XXX。
*   **selected-files & folder 边界**: selected-files 模式下“本地整理输出”与 Native ZIP 强制置灰禁用， folder 模式下的物理整理 copy-only 入口可用且正常导出。
*   **取消扫描行为**: 随时点击“取消”可在秒级返回首页且后台不会发生异步跳转，安全释放资源。
*   **大批量性能表现**: 支持 `30 张 / 100 张 / 200 张` 等多档大批量照片的完整离线分析；虚拟滚动列表在加载 200 张照片时无 DOM 卡顿与排版错乱。

## 5. 已知风险 (Known Risks)

本版本仍为**内部 Alpha 候选阶段**，已知存在以下局限和表现特征，在测试时无需记录为缺陷：

1.  **大内存消耗**: 在分析接近 200 张的高分辨率 JPG/PNG 照片时，内存消耗峰值可能会达到 `3.9 GB`，在此期间低配置电脑可能会有短暂响应延迟。我们将在后续版本中进行 ObjectURL 内存专项回收优化。
2.  **SmartScreen 提示**: 由于未附带签名证书，安装时的拦截窗口属于正常现象。
3.  **HEIC / HEIF 支持有限**: 现阶段本软件的首要目标是保证 JPG/PNG 处理链的绝对稳定。非 JPG/PNG 格式（如 HEIC/HEIF 等）照片可能会在首阶段被分析过滤或跳过。
4.  **构建性能警告**: 前端构建包内仍含有 Next.js 对 `<img>` 标签的加载警告，非阻塞性，不会对桌面端实际运行及响应产生影响。

## 6. 数据与安全边界说明 (Privacy & Security Boundaries)

本软件在交互上严格遵守隐私和物理原图安全红线：

*   **完全本地化**: 照片的所有像素诊断、清晰度评估、相似度聚类及 A/B 挑选均在**用户电脑本地离线完成**，绝不上传任何原图或元数据。
*   **原图绝对安全**: 软件不会对用户的原始照片目录执行任何物理移动、物理删除或物理覆盖操作。
*   **数据脱敏**: 界面不展示真实物理路径，文件名全局脱敏为 `Photo-001` 等掩码格式，保护隐私。
*   **输出控制**: 
    *   在 `selected-files` (选择图片) 模式下，本地物理整理不可用，需先导出整理清单 (JSON/CSV)；
    *   在 `folder` (选择文件夹) 模式下，本地物理整理采用 `copy-only` (复制到新目录) 机制，保留原图完整不受篡改。
*   **Native ZIP**: 目前此功能保持安全禁用。
